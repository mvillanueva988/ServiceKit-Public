# Plan 04-01: Compatibility Guards

## Objective

Auditar y corregir problemas de compatibilidad reales en el toolkit. El foco es
eliminar crashes en entornos Windows 10/11 Home/Pro/LTSC — no agregar complejidad
innecesaria. Entrega: código corregido + matriz de compatibilidad documentada.

## Tasks

### Task 1 — Proteger CIM calls críticos en Get-SystemSnapshot
**Files:** `modules/Telemetry.ps1`

Agregar `-ErrorAction SilentlyContinue` a `Get-CimInstance` de Win32_Processor,
Win32_VideoController, Win32_ComputerSystem, Win32_PhysicalMemory y
Win32_OperatingSystem. Cada resultado null produce un objeto parcial/vacío en lugar
de un crash.

Regla de degradación:
- `Win32_Processor` null → `CPU.Name = 'Unknown'`, Cores/Threads = 0
- `Win32_VideoController` null → `$gpus = @()`
- `Win32_ComputerSystem` null → `$ramTotalGb = 0`
- `Win32_PhysicalMemory` null → `$ramSlots = @()`
- `Win32_OperatingSystem` null → `$uptimeHours = 0`

### Task 2 — Proteger Win32_SystemEnclosure (laptop detection)
**Files:** `modules/Telemetry.ps1`

`Win32_SystemEnclosure` es accedido inline sin guard:
```powershell
[int] $chassisType = (Get-CimInstance -ClassName Win32_SystemEnclosure).ChassisTypes[0]
```
Si falla, el código lanza en el acceso a `.ChassisTypes[0]`. 

Fix: capturar con `-ErrorAction SilentlyContinue`, default `$chassisType = 0` (Desktop/Unknown) si null → salta la detección de batería.

### Task 3 — Proteger Get-PhysicalDisk (Storage module)
**Files:** `modules/Telemetry.ps1`

`Get-PhysicalDisk` no existe en Server Core ni en algunas LTSC mínimas sin el módulo Storage.  
Fix: envolver el bloque `Get-PhysicalDisk` en `try/catch`, resultado vacío → `$disks = @()`.
