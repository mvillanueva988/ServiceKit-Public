<task type="auto">
  <name>Task 1: Crear Launch.ps1 — one-liner handler</name>
  <files>Launch.ps1</files>
  <action>
Crear `Launch.ps1` en la raíz del repo. Este script es el receptor de:
```
irm https://raw.githubusercontent.com/TU_USUARIO/TU_REPO/main/Launch.ps1 | iex
```

**Lógica completa:**

1. Definir constantes al inicio del script con comentario claro marcando qué debe editar el usuario:
   ```powershell
   # ── CONFIGURAR ANTES DE USAR ──────────────────────────────────────────────
   [string] $GitHubRepo  = 'TU_USUARIO/TU_REPO'   # <-- cambiar esto
   [string] $InstallPath = 'C:\PCTk'
   # ─────────────────────────────────────────────────────────────────────────
   ```

2. Intentar obtener el último release via GitHub API:
   ```
   GET https://api.github.com/repos/$GitHubRepo/releases/latest
   ```
   - Con `Invoke-RestMethod -UseBasicParsing -UserAgent 'PCTk-Launcher'`
   - Extraer el asset cuyo `name` termina en `.zip`
   - Si la API falla (sin internet, sin releases publicados aún): ir a paso de fallback

3. Descargar el ZIP a `$env:TEMP\PCTk-update.zip`:
   - Usar `[System.Net.WebClient]::new().DownloadFile($url, $dest)` (más simple que HttpWebRequest para este caso, sin barra de progreso necesaria en el launcher)
   - Mostrar `"Descargando toolkit..."` con `Write-Host`

4. Preservar `tools\bin\` antes de extraer:
   - Si `$InstallPath\tools\bin` existe: mover a `$env:TEMP\PCTk-toolsbin-backup`
   
5. Extraer ZIP a `$InstallPath`:
   - Si `$InstallPath` existe: `Remove-Item $InstallPath -Recurse -Force`
   - `Expand-Archive -Path $zipPath -DestinationPath $InstallPath -Force`
   - Borrar `$env:TEMP\PCTk-update.zip`

6. Restaurar `tools\bin\`:
   - Si el backup existe: mover de vuelta a `$InstallPath\tools\bin`

7. Lanzar:
   ```powershell
   & "$InstallPath\main.ps1"
   ```

**Fallback** (API falló y `$InstallPath\main.ps1` existe):
```powershell
Write-Host '  [!] No se pudo descargar actualizacion. Lanzando version local...' -ForegroundColor Yellow
& "$InstallPath\main.ps1"
```

**Error fatal** (API falló Y no hay versión local):
```powershell
Write-Host '  [!] Sin conexion y sin version local. Descarga el toolkit manualmente.' -ForegroundColor Red
Write-Host "      https://github.com/$GitHubRepo/releases" -ForegroundColor DarkGray
```

**Requisitos de código:**
- `#Requires -Version 5.1`
- `Set-StrictMode -Version Latest`
- No usar `Start-Job` (el launcher es síncrono por diseño — el usuario está esperando)
- No usar `Invoke-WebRequest` para la descarga del ZIP (usa `WebClient.DownloadFile` — más simple, sin parsear headers)
  </action>
  <verify>
    Revisar que el script contiene: `$GitHubRepo`, `$InstallPath = 'C:\PCTk'`, check de fallback, `Expand-Archive`, backup/restore de `tools\bin`, `& "$InstallPath\main.ps1"`.
    Sintaxis: `powershell -NoProfile -Command "& { . .\Launch.ps1 -WhatIf }" 2>&1` — no debe tirar errores de parse.
    En la práctica: `powershell -NoProfile -File .\Launch.ps1` después de configurar `$GitHubRepo`.
  </verify>
  <done>
    Launch.ps1 existe en la raíz. Contiene las constantes con comentario de configuración, lógica de descarga + fallback, preservación de tools\bin\, y lanzamiento de main.ps1. Sin errores de sintaxis PS5.
  </done>

<task type="auto">
  <name>Task 2: Crear Release.ps1 — build script de distribución</name>
  <files>Release.ps1</files>
  <action>
Crear `Release.ps1` en la raíz del repo. Se ejecuta localmente antes de publicar un release en GitHub.

**Parámetros:**
```powershell
param(
    [string] $Version   = (Get-Date -Format 'yyyy.MM.dd'),
    [switch] $Publish   # Si se pasa: sube a GitHub Releases via API (requiere $env:GITHUB_TOKEN)
)
```

**Lógica:**

1. Definir rutas:
   ```powershell
   [string] $source  = $PSScriptRoot
   [string] $staging = Join-Path $env:TEMP 'PCTk-release-staging'
   [string] $outDir  = Join-Path $PSScriptRoot 'dist'
   [string] $zipName = "PCTk-$Version.zip"
   [string] $zipPath = Join-Path $outDir $zipName
   ```

2. Limpiar y crear staging:
   ```powershell
   if (Test-Path $staging) { Remove-Item $staging -Recurse -Force }
   Copy-Item -Path $source -Destination $staging -Recurse
   ```

3. Eliminar del staging los artifacts de desarrollo (directorios y archivos):
   - Directorios a excluir: `.git`, `.gsd`, `Logs`, `output`, `dist`, `memories`
   - Subdirectorio: `tools\bin`
   - Archivos raíz a excluir: `Release.ps1` (el build script no se distribuye), `GSD-STYLE.md`, `CHANGELOG.md`, `*.code-workspace`
   - Usar `Remove-Item -Recurse -Force` sobre cada uno si `Test-Path` lo confirma

4. Crear `dist\` si no existe. Generar ZIP:
   ```powershell
   if (-not (Test-Path $outDir)) { New-Item -ItemType Directory -Path $outDir | Out-Null }
   if (Test-Path $zipPath) { Remove-Item $zipPath -Force }
   Compress-Archive -Path "$staging\*" -DestinationPath $zipPath
   ```
   Nota: `"$staging\*"` (no `$staging`) para que el ZIP no tenga carpeta raíz — la extracción en `Launch.ps1` va directo a `C:\PCTk\` sin subfolder.

5. Limpiar staging:
   ```powershell
   Remove-Item $staging -Recurse -Force
   ```

6. Mostrar resultado:
   ```powershell
   [long] $sizeMB = (Get-Item $zipPath).Length / 1MB
   Write-Host ("  [v] {0}  ({1:N1} MB)" -f $zipName, $sizeMB) -ForegroundColor Green
   Write-Host "      $zipPath" -ForegroundColor DarkGray
   ```

7. Si `-Publish` fue pasado:
   - Verificar `$env:GITHUB_TOKEN` — si no existe, mostrar error y salir
   - Leer `$GitHubRepo` desde `Launch.ps1` (grep de la línea `$GitHubRepo = '...'`) o desde un parámetro `-Repo`
   - Crear release via `POST https://api.github.com/repos/$repo/releases` con `tag_name = "v$Version"`
   - Subir el ZIP asset via `POST $uploadUrl?name=$zipName` con el body del archivo
   - Si falla la API: mostrar error con instrucción manual de subir el ZIP en GitHub UI

**Requisitos:**
- `#Requires -Version 5.1`, `Set-StrictMode -Version Latest`
- El output del script solo va a `Write-Host` (no a Write-Output) para no contaminar pipelines
- `dist\` debe estar en `.gitignore` (verificar y agregar si no está)
  </action>
  <verify>
    Ejecutar: `.\Release.ps1` (sin -Publish).
    Verificar que se genera `dist\PCTk-YYYY.MM.DD.zip`.
    Verificar que el ZIP NO contiene: `.git/`, `.gsd/`, `Logs/`, `output/`, `tools/bin/`, `Release.ps1`.
    Verificar que el ZIP SÍ contiene: `main.ps1`, `Run.bat`, `modules/`, `utils/`, `core/`, `tools/manifest.json`.
    `Compress-Archive` + `Expand-Archive` round-trip: extraer el ZIP a un temp dir y confirmar estructura correcta.
  </verify>
  <done>
    Release.ps1 existe. Genera ZIP limpio en dist/. El ZIP tiene estructura plana (sin carpeta raíz). dist/ agregado a .gitignore. Sin errores de sintaxis PS5.
  </done>
