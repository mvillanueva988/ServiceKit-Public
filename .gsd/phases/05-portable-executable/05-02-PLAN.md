El bug: `Test-ToolInstalled` devuelve `$true` si el directorio/archivo existe, sin verificar si está completo. Una descarga parcialmente fallida queda en disco y nunca se re-descarga.

**Modificar `Test-ToolInstalled`** para agregar validación de tamaño usando `approxSizeMB` del manifest:

Para herramientas ZIP (check por `launchExe`):
```powershell
function Test-ToolInstalled {
    param([object] $Tool)
    $isZip = ($Tool.PSObject.Properties['type'] -and $Tool.type -eq 'zip') -or
             ($Tool.filename -like '*.zip')

    if ($isZip) {
        # Para ZIPs, verificar que el launchExe exista dentro del extractDir
        if ($Tool.PSObject.Properties['launchExe'] -and $Tool.launchExe) {
            $launchPath = Join-Path $binDir $Tool.launchExe
            return Test-Path $launchPath
        }
        if ($Tool.PSObject.Properties['extractDir'] -and $Tool.extractDir) {
            return Test-Path (Join-Path $binDir $Tool.extractDir)
        }
    }

    # Para EXEs: verificar existencia Y tamaño mínimo
    $exePath = Join-Path $binDir $Tool.filename
    if (-not (Test-Path $exePath)) { return $false }

    # Si el manifest declara approxSizeMB, verificar que el archivo tenga al menos 50% de ese tamaño
    if ($Tool.PSObject.Properties['approxSizeMB'] -and $Tool.approxSizeMB -gt 0) {
        [long] $minBytes = [long]($Tool.approxSizeMB * 0.5 * 1MB)
        [long] $actual   = (Get-Item $exePath).Length
        if ($actual -lt $minBytes) {
            Write-Host ("  [!] {0}: archivo incompleto ({1:N1} MB de ~{2} MB esperados), forzando re-descarga" -f `
                $Tool.name, ($actual / 1MB), $Tool.approxSizeMB) -ForegroundColor Yellow
            return $false
        }
    }

    return $true
}
```

Lógica del cambio:
- ZIP con `launchExe` → verificar que el ejecutable dentro del ZIP extraído exista (más preciso que solo el directorio)
- EXE con `approxSizeMB` → verificar que el tamaño sea ≥ 50% del esperado. 50% es conservador: un EXE de 50MB debe tener al menos 25MB, ruling out archivos de 0-5MB que son claramente parciales
- Sin `approxSizeMB` → comportamiento anterior (solo existencia), no rompe herramientas sin ese campo en el manifest

No modificar nada más en Bootstrap-Tools.ps1 — solo la función `Test-ToolInstalled`.

Toolkit de optimización y diagnóstico para Windows 10/11. Diseñado para uso técnico — servicio a PC de clientes, tanto local como vía AnyDesk.

Abrir PowerShell como administrador y ejecutar:

irm https://raw.githubusercontent.com/TU_USUARIO/TU_REPO/main/Launch.ps1 | iex

## Qué hace

- **Optimización**: Servicios bloat, temporales, registro, privacidad, telemtería

- **Mantenimiento**: Windows Update, chkdsk, SFC, puntos de restauración
- **Performance**: Perfiles de energía, startup apps, RAM

## Uso sin one-liner (método manual)

1. Descargar el último release: [Releases](https://github.com/TU_USUARIO/TU_REPO/releases)

## Requisitos

- Windows 10 / Windows 11
- PowerShell 5.1 (incluido en Windows)
- Permisos de administrador
