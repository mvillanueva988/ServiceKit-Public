PC Optimizacion Toolkit — Sesión de Desarrollo
Resumen de cambios

Segunda sesión de desarrollo. Foco principal: sistema de auditoría diagnóstica PRE/POST service.
Se analizaron los scripts de la versión anterior (oldscripts/) para rescatar lógica válida y descartar lo que causaba bugs y exceso de información.

Archivos del repositorio anterior analizados como referencia
Archivo	Descripción
collect_info.ps1	Recopilación de datos del sistema (20 pasos síncronos, sin jobs)
compare_pre_post.ps1	Comparación PRE/POST basada en mover JSONs manualmente entre carpetas
generate_report.ps1	Generador de reportes Markdown + HTML desde JSON

Problemas identificados en la versión anterior
- Flujo PRE/POST roto: el técnico debía mover manualmente el JSON de output\pre_service\ a output\post_service\. Sin ese paso manual el comparador fallaba con error.
- Velocidad: 20 pasos totalmente síncronos. Get-WindowsDriver -Online -All podía tardar 60+ segundos. Microsoft.Update.Session COM era lento e inconsistente.
- Exceso de datos: capturaba slots de RAM, BIOS, placa madre, event log de 7 días, scheduled tasks completas, 300+ apps instaladas — ninguno de esos datos cambia con un service.
- Temperatura CPU: tenía un placeholder vacío con el comentario "HWiNFO CLI requiere versión PRO".

Lógica rescatada del código viejo
- Get-StorageReliabilityCounter para temperatura, wear% y errores SMART de discos físicos.
- Detección GPU dedicada/integrada por regex en el nombre del adaptador.
- Estructura de comparación de volúmenes por letra de unidad (join key: DriveLetter).
- Cálculo de battery health: (FullChargeCapacity / DesignCapacity) * 100.
- Score "X/N áreas mejoradas" del reporte comparativo — pieza más valiosa para el técnico.

[modules/Telemetry.ps1] — NUEVO

Módulo de auditoría diagnóstica completo siguiendo la arquitectura asíncrona del proyecto.

Get-SystemSnapshot -Phase Pre|Post
  Recopila vía CIM/WMI:
  - CPU: nombre, cores, threads
  - GPU: nombre, tipo (Dedicated/Integrated por regex), versión de driver
  - RAM: total GB + slots (capacity, speed, manufacturer) — excluye slots vacíos
  - Discos físicos: health status, temperatura SMART, wear%, errores lectura/escritura
  - Volúmenes: espacio libre GB y porcentaje usado por letra de unidad
  - Page File: CurrentUsageMb y PeakUsageMb (Win32_PageFileUsage)
  - Servicios: running count + lista de bloat activos
  - Startup count: entries de registry Run + carpetas de inicio (sin scheduled tasks)
  - Top 5 procesos por WorkingSet en MB
  - Batería: charge%, health%, status (solo laptops, detectado por ChassisType)
  - Antivirus: lista desde SecurityCenter2 + Windows Defender, flag MultipleAvProblem
  - Temperatura CPU: best-effort via MSAcpi_ThermalZoneTemperature (root/wmi), $null si no disponible
  - UptimeHours para detectar si el sistema fue reiniciado entre PRE y POST
  Retorna PSCustomObject tipado con campo Phase incorporado.

Save-Snapshot -Phase Pre|Post
  Ejecuta Get-SystemSnapshot y guarda el resultado como JSON en output\snapshots\.
  Nombre de archivo: yyyy-MM-dd_HHmmss_pre.json / yyyy-MM-dd_HHmmss_post.json.
  Retorna Phase, FilePath y FileName. Sin movidas manuales de archivos.

Compare-Snapshot [-PrePath] [-PostPath]
  Carga automáticamente los JSONs más recientes de cada fase desde output\snapshots\.
  Genera diff estructurado con:
  - Espacio liberado por volumen y total
  - Delta de servicios en ejecución y bloat deshabilitado
  - Delta de programas de inicio
  - Diff de batería (solo laptops)
  - Flag de conflicto de AV resuelto
  - Flag de reinicio detectado
  - Score 0/6 con lista de mejoras concretas

Show-SnapshotComparison -Diff $diff
  Visualización coloreada en consola:
  - [+] verde = área mejorada
  - [-] rojo = área empeorada
  - [ ] gris = sin cambios
  - [!] amarillo = advertencia (ej: no reiniciado)
  Score final coloreado: verde ≥5, amarillo ≥3, rojo <3.

Start-TelemetryJob -Phase Pre|Post
  Wrapper asíncrono sobre Save-Snapshot. Dot-sourcea el módulo dentro del job
  para garantizar acceso a todas las funciones en el contexto del background job.
